Technical evidence

What the model proves, and what it still has to earn.

The current canvas is a concept replay, not a claimed field result. The pilot exists to replace hand-tuned assumptions with calibrated Bengaluru observations and repeatable baseline comparisons.

State variables rho, q, Pi_m, wait, spillback risk

Density and flow are visualized with an LWR-inspired fundamental diagram; Pi_m discounts discharge during idle and acceleration after a red phase.

Objective Reduce sector delay without starving cross movement

The current replay scores queue pressure, wait, synthetic downstream risk, and minimum green; the field pilot adds pedestrian, emergency, stale-data, and operator fallback constraints.

Baseline Fixed-time and actuated comparison

Every replay should produce a before/after trace: current plan, recommended phase, expected effect, confidence, and failure reason.

Calibration Field labels before field control

Queue length, discharge rate, spillback, empty-green, phase state, and weather/event tags become the acceptance dataset.

Current limit Synthetic context pressure

Neighbor-node pressure in the demo is synthetic until replaced with measured queues, occupancy, and downstream capacity labels.

Validation path

Start with replay. Earn trust before control.

PATC is intentionally a shadow-mode decision-support layer first. The pilot should run observed replay across a 4-6 signal sector through peak, off-peak, rain, weekend, school, event, and seasonal variation before any deployment request.

Observed

Cross-junction failure

One upstream signal can clear while the next road stays saturated. The system is built around that linked-sector failure mode.

Modeled

Flow plus startup loss

The product tracks density, discharge, Pi_m, phase timing, wait, and pressure instead of treating every junction as isolated.

Simulated

Hypothesis targets

The percentage numbers are simulation targets that must be validated against fixed-time and actuated baselines.
