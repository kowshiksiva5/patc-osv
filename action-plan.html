Probabilistic Adaptive Traffic Control (PATC)

Traffic lights that predict movement, not just count cars.

OSV Fellowship 2026
12-month action plan
PATC-AP-2026-001

The problem: Fixed-time and detector-based signals waste green time because they ignore arrival uncertainty and driver start-up delay. Most global cities lose 120–150 hours per driver annually to congestion (TomTom Traffic Index). Bengaluru alone costs an estimated ₹20,000 crore/year in lost productivity. PATC treats a corridor as a moving pressure field — modeling density, discharge, startup loss, and spillback risk — to coordinate signals that clear congestion instead of displacing it.

The Approach

Core Model Stack

  • LWR conservation law — discretized density field per link
  • Markov mobility model — Idle → Accelerating → Flow states with probabilistic transition rates
  • Poisson/NB arrival forecast — EWMA-updated stochastic arrival rate
  • MPC optimizer — rolling horizon QP with delay, queue, spillback, and safety penalties
  • Particle/Kalman filter — online state estimation under sensor sparsity

Key Differentiators

  • Startup-loss damping (Πₘ) prevents over-serving phases where queues haven't woken up
  • Sector-level spillback detection before queues lock up — not post-hoc correction
  • Every recommendation: action + reason + confidence + fallback + replay trace
  • Shadow mode first — 40–60 clean site-hours before any live actuation
  • Safety-hard: min greens, pedestrian windows, emergency priority, stale-data stops

12-Month Roadmap

Months 1–3

Foundation & Model Development

  • Implement LWR-CTM in Python/NumPy
  • Build Markov chain mobility module + MLE calibration
  • Prototype EWMA arrival estimator
  • Establish 4 SUMO baseline networks (isolated, arterial, grid, hybrid)
  • Begin field observation on target corridor
Months 4–6

MPC Integration & Co-simulation

  • Integrate OSQP-based QP solver into MPC loop
  • Implement TraCI interface for real-time SUMO co-simulation
  • Validate state estimation accuracy (Kalman filter metrics)
  • Tune MPC weights via grid search on baseline network
  • Launch public demo website with corridor replay
Months 7–9

Simulation Campaign & Analysis

  • 30-replication Monte Carlo campaigns on all four networks
  • Comparative KPI tables with Welch t-test significance
  • Emission & fuel savings reports (HBEFA3 baseline)
  • Sensitivity analyses on λ₁, λ₂, μ, H, Nₓ parameters
  • Publish open-source benchmark report
Months 10–12

Edge Prototype & Pilot Proposal

  • Port MPC to embedded target (RPi 5 / Jetson Orin NX)
  • Implement NTCIP 1202 communication interface
  • Hardware-in-the-loop testing against SUMO live feed
  • Prepare field trial protocol for target corridor
  • Submit pilot proposal to transport authority

Key Risks & Mitigations

  • Simulation ≠ reality → Early field observation (month 3+)
  • Sensor sparsity → Particle filter + conservative fallbacks
  • Extreme events → Chance constraints + auto fixed-time revert
  • Operator trust → Replay traces + feedback from shadow day 1
  • Bureaucratic delay → Build evidence first, negotiate second
