Rolling horizon view

The recommender scores the near-future consequence of a timing change instead of reacting only to the present queue count.

System architecture

Three layers, one operator-reviewed decision.

01 Observe

Field observations, signal phase state, queue length, discharge rate, weather, and local anomaly tags.

02 Estimate

Queue pressure, arrival forecast, movement probability, downstream spillback risk, and confidence score.

03 Recommend

Timing or offset action with reason, expected effect, and fallback trigger for shadow-mode review.

Validation

Prove the model before any controller integration.

Shadow replay

Re-run observed periods against fixed-time, local-actuated, and PATC recommendations to measure whether the system would have made better choices.

Simulation stress tests

Build corridor scenarios for rain, office surge, downstream blockage, pedestrian pressure, and seasonal traffic variation.

The first real target is not autonomous control. It is a trusted, explainable shadow-mode prototype that operators can inspect and reject.
